## 四、内容规范

### 4.1 章节结构模板

```markdown
# 章节标题

> 一句话概述

## 前言：为什么这个话题重要？
（背景介绍，问题引入）

## 第一部分：核心概念
### 一、xxx
### 二、xxx

## 第二部分：具体方法
### 一、xxx
### 二、xxx

## 第三部分：实践指南
（可操作的建议）

## 常见问题

## 快速参考卡
（要点总结，方便查阅）

## 参考资料

---
*上一篇：[xxx](./prev.md)*
*下一篇：[xxx](./next.md)*
```

### 4.2 格式规范

| 元素 | 规范 |
|------|------|
| 标题层级 | 最多到 H4，避免 H5 |
| 列表 | 使用 `-` 或数字，保持一致 |
| 表格 | 表头必须有，对齐整齐 |
| 代码块 | 指定语言，如 `markdown` 或 `text` |
| 强调 | 使用 `**粗体**` 强调重点 |
| 引用 | 使用 `>` 引用重要信息 |

### 4.3 写作风格

- 语气：专业但亲切，像朋友在分享经验
- 受众：零基础到进阶跑者都能读懂
- 实用性：每个章节都要有可操作的建议
- 科学性：重要观点要有依据或标注来源
